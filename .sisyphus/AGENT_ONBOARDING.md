# 🤖 Agent Onboarding Checklist

**For any agent starting work on this project**

---

## ✅ Pre-Work Checklist

Before starting ANY work, complete these steps:

- [ ] Read `/home/ubuntu/bench/pro-secretary/TASK.md` (main handoff document)
- [ ] Read `/home/ubuntu/bench/pro-secretary/.sisyphus/RULES.md` (update protocol)
- [ ] Understand current active tasks
- [ ] Check for blockers
- [ ] Review recently completed work

---

## ✅ During Work

- [ ] Follow existing code patterns
- [ ] Document decisions as you go
- [ ] Note any gotchas discovered
- [ ] Track files changed

---

## ✅ Post-Work Checklist (MANDATORY)

Before reporting completion to user:

- [ ] Work is complete and verified
- [ ] Tests pass (if applicable)
- [ ] TASK.md updated with:
  - [ ] Completed items marked ✅
  - [ ] Moved to "Recently Completed" with timestamp
  - [ ] Files changed listed
  - [ ] "Communication Notes" entry added
  - [ ] "Last Updated" timestamp updated
  - [ ] New blockers documented (if any)
  - [ ] "NEXT STEPS" re-prioritized
- [ ] Changes committed (if applicable)
- [ ] Ready for next agent handoff

---

## 🚫 Common Mistakes to Avoid

1. ❌ Starting work without reading TASK.md
2. ❌ Completing work without updating TASK.md
3. ❌ Not documenting decisions
4. ❌ Leaving blockers undocumented
5. ❌ Not listing files changed

---

## 🎯 Success Criteria

You've successfully completed your work when:

1. ✅ Original task is done
2. ✅ TASK.md is updated
3. ✅ Next agent can continue seamlessly
4. ✅ User has visibility into progress

---

**Remember:** TASK.md update is NOT optional. It's part of task completion.
